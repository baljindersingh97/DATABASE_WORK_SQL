# Design and Implementation of a Normalized Relational Database System for a Hotel Reservation and Guest Management Application Using SQL

## Project Overview

This project presents the design and implementation of a **Hotel Reservation and Guest Management System** using **SQL** and **MySQL Workbench**. The system was developed as a relational database solution to manage core hotel operations in a structured and efficient way.

The database supports the storage and management of customer information, room types, rooms, reservations, payments, hotel services, staff details, and reservation status history. It was designed using normalization principles, relational integrity constraints, and indexing strategies to improve consistency, maintainability, and query performance.

The project also demonstrates full **CRUD operations** and a set of **advanced SQL queries** for business reporting and operational analysis.

---

## Project Objectives

The main objectives of this project are:

- To design a normalized relational database schema for hotel operations
- To implement the database using SQL
- To enforce data integrity using primary keys, foreign keys, unique constraints, check constraints, and default values
- To populate the database with realistic sample data
- To demonstrate CRUD operations
- To perform advanced SQL queries for reporting and analysis

---

## System Scope

The system is designed for a **single hotel branch** and supports the following functions:

- customer management
- room type and room management
- reservation management
- payment tracking
- service allocation to reservations
- staff-based reservation status tracking
- reporting and analytical queries

### Out of Scope

The following features are not included in this project:

- hotel website or front-end booking interface
- multi-branch hotel management
- online payment gateway integration
- loyalty or rewards system
- housekeeping workflow management
- role-based login system
